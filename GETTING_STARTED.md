# Getting Started with IRONKANG

A simple guide to using the new build system.

---

## 🚀 First Time Setup

When you first get the project, you need to install dependencies:

```bash
npm install
```

**What this does:** Downloads Vite and other tools needed for building the app.

**You only need to do this once** (or after pulling new changes that update `package.json`).

---

## 💻 Development (Working on the App)

To work on the app and see your changes live:

```bash
npm run dev
```

**What happens:**
1. A development server starts
2. Your browser opens automatically to `http://localhost:3000`
3. You can now edit files and see changes instantly
4. Press `Ctrl+C` in the terminal to stop

**Example workflow:**
```bash
# Start development server
npm run dev

# Browser opens automatically
# Make changes to index.html
# Save the file
# Browser automatically refreshes with your changes!

# When done, press Ctrl+C to stop
```

---

## 🏗️ Building for Production

When you're ready to deploy or share your app:

```bash
npm run build
```

**What happens:**
1. Vite processes your files
2. Minifies HTML/CSS/JS
3. Creates optimized files in the `dist/` folder
4. Takes about 100-200ms

**The `dist/` folder contains:**
```
dist/
├── index.html          # Minified HTML
├── gifs/              # All exercise videos
├── icon-192.png       # PWA icons
├── icon-512.png
├── manifest.json      # PWA manifest
└── sw.js             # Service worker
```

**This is what you deploy to your web server!**

---

## 👀 Preview Production Build

Want to test the production build locally before deploying?

```bash
npm run preview
```

**What happens:**
1. Starts a local server for the `dist/` folder
2. Opens `http://localhost:4173`
3. You can test the production version locally

**Note:** You must run `npm run build` first!

---

## 📦 Complete Workflow Example

Here's a typical development session:

```bash
# 1. Install dependencies (first time only)
npm install

# 2. Start development
npm run dev
# Browser opens, make your changes...

# 3. When satisfied, build for production
npm run build

# 4. Test production build locally
npm run preview
# Check that everything works

# 5. Deploy the dist/ folder to your server
# (Upload dist/ contents to GitHub Pages, Netlify, etc.)
```

## 🔄 Differences from Before

### Before (Old Way)
- Edit `index.html` directly
- Refresh browser manually to see changes
- Upload `index.html` + all files to server
- No optimization

### Now (New Way)
- Edit `index.html` in root folder
- Changes appear automatically (hot reload)
- Run `npm run build`, upload only `dist/` folder
- Files are optimized and minified

## 📁 What Files to Edit

**Files you work on:**
- `index.html` - Your main app file
- `public/gifs/*.mp4` - Exercise videos
- `public/manifest.json` - PWA settings
- `public/sw.js` - Service worker

**Files you DON'T edit:**
- `dist/*` - Generated by build, don't edit these
- `node_modules/*` - Dependencies, auto-generated

## 🚨 Common Questions

### Q: Do I need to run `npm run build` while developing?
**A:** No! Use `npm run dev` for development. Only build when you're ready to deploy.

### Q: What's the difference between `dev` and `build`?
**A:**
- `dev` = Development server for making changes (fast, not optimized)
- `build` = Create production files for deployment (slow, optimized)

### Q: Where do I upload for deployment?
**A:** Upload everything inside the `dist/` folder to your web host.

### Q: I made changes but don't see them
**A:**
- If using `npm run dev`: Save the file, changes appear instantly
- If using `npm run preview`: Run `npm run build` again first

### Q: Can I still open index.html directly in browser?
**A:** Not recommended. The paths won't work correctly. Always use `npm run dev`.

### Q: What if I get an error about "vite not found"?
**A:** Run `npm install` first.

## 🆘 Troubleshooting

### Error: "command not found: npm"
You need to install Node.js first: https://nodejs.org/

### Error: "Cannot find module 'vite'"
Run `npm install` to install dependencies.

### Port 3000 already in use
Another app is using port 3000. Either:
- Close that app
- Or edit `vite.config.js` and change the port number

### Changes not appearing
1. Make sure you saved the file
2. Check the terminal for errors
3. Try refreshing the browser (F5)
4. If still stuck, stop server (Ctrl+C) and run `npm run dev` again
